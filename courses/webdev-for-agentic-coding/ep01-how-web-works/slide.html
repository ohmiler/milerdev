🌍 Internet vs Web
🔌

Internet

เครือข่ายคอมพิวเตอร์ขนาดใหญ่ทั่วโลกที่เชื่อมต่อกัน — เหมือน "ถนน" ที่ข้อมูลวิ่งผ่าน

🌐

Web (World Wide Web)

"บริการ" หนึ่งที่ทำงานบน Internet — เว็บไซต์ที่เราเปิดดูผ่าน Browser

💡 Web เป็นแค่ส่วนหนึ่งของ Internet — Email, Chat, Game Online ก็ใช้ Internet เหมือนกัน

📡 HTTP / HTTPS คืออะไร

HTTP = HyperText Transfer Protocol — "ภาษา" ที่ Client กับ Server ใช้คุยกัน

🔓 HTTP

ส่งข้อมูลแบบไม่เข้ารหัส — ใครดักจับได้ก็อ่านได้

🔒 HTTPS

เหมือน HTTP แต่เข้ารหัส (SSL/TLS) — ปลอดภัยกว่า ใช้กันเป็นมาตรฐาน

✅ ปัจจุบันเว็บส่วนใหญ่ใช้ HTTPS — สังเกตจากรูปกุญแจ 🔒 ใน Address Bar

📋 HTTP Methods ที่ใช้บ่อย
Method ทำอะไร ตัวอย่าง
GET ขอข้อมูล (อ่าน) เปิดหน้าเว็บ, ดูรูปภาพ
POST ส่งข้อมูล (สร้างใหม่) สมัครสมาชิก, โพสต์ข้อความ
PUT อัปเดตข้อมูล (แก้ไข) แก้ไขโปรไฟล์
DELETE ลบข้อมูล ลบโพสต์, ลบบัญชี
🚦 HTTP Status Codes

Server ตอบกลับมาพร้อม "รหัสสถานะ" บอกว่าสำเร็จหรือไม่

2xx — สำเร็จ ✅

200 OK — ทุกอย่างปกติ
201 Created — สร้างสำเร็จ

3xx — Redirect 🔀

301 — ย้ายถาวร
302 — ย้ายชั่วคราว

4xx — Client Error ❌

404 Not Found — ไม่เจอ
403 Forbidden — ไม่มีสิทธิ์

5xx — Server Error 💥

500 — Server พัง
503 — Service ไม่พร้อม

🔗 โครงสร้างของ URL
https://www.example.com:443/products/shoes?color=red&size=42#reviews

Protocol

https:// — วิธีการสื่อสาร

Domain

www.example.com — ที่อยู่เว็บ

Path

/products/shoes — เส้นทางไปหน้าที่ต้องการ

Query

?color=red — ข้อมูลเพิ่มเติม

💡 DNS (Domain Name System) เปรียบเหมือนสมุดโทรศัพท์ — แปลงชื่อที่คนจำง่าย ให้เป็นตัวเลขที่คอมพิวเตอร์เข้าใจ

🎨 Browser แสดงเว็บไซต์อย่างไร
  1. Browser ส่ง HTTP Request ไปยัง Server
  2. Server ส่ง HTML กลับมา
  3. Browser อ่าน HTML แล้วสร้าง DOM (โครงสร้างของหน้า)
  4. โหลดไฟล์ CSS แล้วสร้าง Style Rules
  5. รวม DOM + CSS เป็น Render Tree
  6. โหลดและรัน JavaScript
  7. Paint — วาดหน้าเว็บให้เราเห็น!
🏗️ สามเสาหลักของเว็บ
📄

HTML

โครงสร้าง — กำหนดว่ามีอะไรบ้างในหน้า (ข้อความ, รูป, ปุ่ม)

เปรียบเหมือน โครงกระดูก

🎨

CSS

รูปลักษณ์ — กำหนดว่าหน้าตาเป็นอย่างไร (สี, ขนาด, ตำแหน่ง)

เปรียบเหมือน เสื้อผ้า/การแต่งตัว

⚡

JavaScript

พฤติกรรม — ทำให้หน้าเว็บมีชีวิต (คลิก, แสดง/ซ่อน, โหลดข้อมูล)

เปรียบเหมือน สมองและกล้ามเนื้อ

🤖 ทำไมต้องรู้สิ่งเหล่านี้ก่อนใช้ AI?
  • อ่านโค้ดที่ AI สร้างให้ได้ — ต้องรู้ว่า HTML, CSS, JS ทำอะไร
  • สั่ง AI ได้ตรงจุด — รู้ศัพท์เทคนิค เช่น "เพิ่ม header", "ทำ responsive"
  • Debug ได้ — เข้าใจ HTTP errors, เปิด DevTools ดู Network ได้
  • ตัดสินใจได้ — เลือก framework, เข้าใจ trade-offs
  • ไม่ถูกหลอก — รู้ว่าโค้ดที่ AI เขียนมา ถูกหรือผิด

⚠️ AI เป็นเครื่องมือที่ทรงพลัง แต่ถ้าไม่มีพื้นฐาน — คุณจะไม่รู้ว่ามันเขียนผิดตอนไหน
