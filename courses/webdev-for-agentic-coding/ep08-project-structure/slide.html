โครงสร้างโปรเจกต์

จัดระเบียบโค้ดอย่างมืออาชีพ — ให้ทั้งคนและ AI เข้าใจ

🤔 ทำไมโครงสร้างถึงสำคัญ
  • หาไฟล์ง่าย — รู้ว่าอะไรอยู่ที่ไหน
  • ทำงานเป็นทีมได้ — ทุกคนเข้าใจตรงกัน
  • Scale ได้ — โปรเจกต์โตขึ้นไม่พัง
  • AI เข้าใจง่าย — Agentic Coding ทำงานได้ดีขึ้นเมื่อโครงสร้างชัดเจน
  • Convention over Configuration — Framework มีมาตรฐานที่ทุกคนเข้าใจ

⚙️ Config Files ที่พบบ่อย
ไฟล์ทำอะไร
package.jsonDependencies & scripts ของโปรเจกต์
tsconfig.jsonตั้งค่า TypeScript
tailwind.config.jsตั้งค่า Tailwind CSS
.eslintrc.jsonกฎ Linting — ตรวจโค้ด
.prettierrcกฎ Formatting — จัดรูปแบบโค้ด
.gitignoreไฟล์ที่ Git ไม่ต้อง track
.envEnvironment variables (ห้าม commit!)
README.mdเอกสารอธิบายโปรเจกต์
📂 โฟลเดอร์ที่พบบ่อยใน Web Projects

src/

Source code หลัก — โค้ดที่เราเขียน

public/

ไฟล์ static — รูปภาพ, fonts, favicon

components/

UI Components ที่ใช้ซ้ำได้ — Button, Card, Modal

lib/ หรือ utils/

Helper functions, utilities

styles/

CSS / Tailwind styles

api/

API routes / Backend logic

📝 README.md — เอกสารสำคัญที่สุด
# Project Name

คำอธิบายสั้นๆ ว่าโปรเจกต์นี้คืออะไร

## Getting Started

```bash
npm install
npm run dev
```

## Tech Stack
- Next.js 14
- TypeScript
- Tailwind CSS

## Project Structure
อธิบายโครงสร้างโฟลเดอร์สำคัญ

✅ README ที่ดี = คนใหม่ (หรือ AI) อ่านแล้วเริ่มทำงานกับโปรเจกต์ได้ทันที

📦 Dependencies — Library ที่โปรเจกต์ใช้

dependencies

ต้องใช้ใน production

"dependencies": {
  "next": "^14.0.0",
  "react": "^18.0.0",
  "axios": "^1.6.0"
}

devDependencies

ใช้เฉพาะตอน develop

"devDependencies": {
  "typescript": "^5.0.0",
  "eslint": "^8.0.0",
  "prettier": "^3.0.0"
}

💡 npm install ติดตั้ง library ที่ระบุใน package.json → เก็บใน node_modules/

🤖 โครงสร้างที่ AI ชอบ
  • ตั้งชื่อไฟล์สื่อความหมาย — UserProfile.tsx ไม่ใช่ comp1.tsx
  • แยกไฟล์ตาม feature — ไม่ยัดทุกอย่างในไฟล์เดียว
  • มี README ที่ชัดเจน — AI อ่านแล้วเข้าใจโปรเจกต์ทันที
  • ใช้ TypeScript — มี type information ช่วย AI เข้าใจโค้ด
  • โครงสร้างเป็นมาตรฐาน — ใช้ convention ของ framework

⚠️ โครงสร้างที่ดี = AI สร้างโค้ดที่ถูกต้องและเข้ากับโปรเจกต์ได้ง่ายกว่า
